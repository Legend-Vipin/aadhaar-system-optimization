# Project Structure & Architecture

## Directory Layout
The project is organized to separate concerns between code, data, and presentation.

### `/` (Root)
- `submission_runner.py`: The main entry point for reviewers. Orchestrates the full analysis.
- `analysis_date_holiday.py`: Specialized module for holiday impact analysis.
- `README.md`: Project overview and usage instructions.

### `src/` (Core Logic)
- `config.py`: Centralized configuration (paths, constants).
- `data_loader.py`: Responsibility: IO operations and Data Cleaning.
- `analytics.py`: Responsibility: ML models, Statistics, Plotting.
- `reporting.py`: Responsibility: PDF Report assembly.
- `main.py`: Legacy pipeline entry point.

### `outputs/` (Generated Artifacts)
- `figures/`: All PNG plots generated by `analytics.py`.
- `reports/`: Final consolidated PDF reports.

### `docs/` (Documentation)
- `methodology.md`: Explanation of algorithms and data flow.
- `conclusion.md`: Strategic insights and summary.
